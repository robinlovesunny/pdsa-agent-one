# 账户

## Credits

### 什么是 Qoder Credits？

用户无法无限制地使用高级大语言模型，因此我们采用基于 Credits 的配额机制。Credits 表示 AI 在执行任务时消耗的资源配额。在 Qoder 中，以下用户请求会消耗 Credits：

- Inline Chat
- Ask 模式
- Agent 模式
- Quest Mode
- Repo Wiki
请注意，具体消耗的 Credits 数量会因任务复杂度和所使用的特定高级模型而有所不同。关于不同任务的 Credits 消耗明细，请参阅我们的任务与 Credits 消耗指南。

### 如何扣减 Credits？
不同时间获取的 Credits 到期时间各不相同。系统会优先使用最先到期的 Credits，帮助您最大化发挥 Credits 的价值。

即使用户已用尽高级模型的配额（即 Credits 已耗尽），我们仍会提供每日限量的基础模型调用额度，确保用户可不间断地使用产品的核心功能。

### 错误
失败的 Qoder 模型请求不会扣减 Credits。只有在模型 API 调用成功时才会扣除 Credits。

### 任务与 Credits 消耗指南
不同任务需要使用不同的模型，每种模型都有相应成本。为此，我们分析了用户在线活动的资源消耗。基于这些数据，我们建立了各类模型使用与 Credits 消耗之间的对应关系，帮助用户更清晰地了解我们如何对不同操作计算并扣除 Credits。

例如，单次 Ask 用户请求可能会使用高级大型语言模型，消耗一定数量的输入与输出 Token。扣除的 Credits 将依据所用模型以及输入与输出的 Token 总量计算。当用户发起 Agent 请求时，通常会在后台触发多次高级模型调用，因此比简单的 Ask 请求消耗的资源更多，Quest Mode 同理。对于 Repo Wiki，其资源消耗将根据为特定代码仓库生成知识库所需的模型推理量来衡量。

表格中的数值为基于统计分析的估算。实际的 Credits 扣减可能有所差异，并以实时使用为准。我们也在持续优化，以在完成同等工作的前提下减少资源消耗。

| 功能 | 50K Context Window | 200K Context Window |
|------|-------------------|---------------------|
| Ask | ~ 3 Credits / User Request | ~ 4 Credits / User Request |
| Agent | ~ 7 Credits / User Request | ~ 12 Credits / User Request |
| Quest | / | ~ 100 Credits / Quest Task |
| Repo Wiki | / | ~ 50 Credits / Repository |

**注意：** 未来随着新功能推出，我们可能会依据其资源需求更新 Credits 消耗率。这意味着某些操作的 Credits 扣减数可能会调整，以更准确反映新功能的实际资源使用情况。

### 使用
您可以在 Usage 仪表板查看您的 Credits 历史和当前用量。

#### 查看你的 Credits 使用情况
1. 登录 Qoder 官网，点击右上角头像，进入 **Settings > Usage**
2. 在这里，你可以查看当前 Plan，以及资源包中可用与已使用的 Credits 信息

**Credits 使用优先级：** 我们会优先消耗最先到期的 Credits。对于到期时间相同的 Credits，将先使用 Plan Credits，再使用 Add-on Credits。

**Credits 到期：** 不同类型的 Credits 有各自的到期时间。你可以在 Credits log 中查看理论到期时间。（请注意，实际到期可能因账户升级等因素而提前。更多详情请参阅 Credits 日志部分。）

通过你的 Plan 获得的 Credits 在当前订阅周期内有效。这些 Credits 会在订阅周期结束时自动归零。

#### Credits 日志：Credits 历史与追踪
Credits 获取历史提供了所有已获得 Credits 的完整记录，包括获取原因、数量，以及生效与到期日期，便于你轻松追踪 Credits 来源。

**关于到期日期的重要说明：**

- Credits 日志中显示的到期日期基于发放时的初始预估。实际到期日期可能会因账户升级或其他因素而发生变化。要确定准确的到期日期，你需要参考你的套餐（Plan）变更历史。
- Credits 日志仅显示 Credits 的获取历史，要查看你当前可用的 Credits 余额，请参阅你账户仪表盘中的"Credits 使用"卡片。

### 我应该选择哪个方案？
- 如果你主要使用代码补全和基础的 Agent 辅助，**Pro 方案**非常适合你的日常开发。
- 如果你高度依赖智能体进行自主编码，**Pro+ 方案**更能满足你更高强度的需求。
- 如果你经常运行 Quest 这类长时间的任务，**Ultra 套餐**中的大量资源更适合你。

#### 查看你的 Credits 使用情况
你可以随时在个人设置中的 Usage 页面查看当前订阅周期内的 Credits 使用情况。不同功能的 Credits 消耗速率各不相同。详情请参阅 Credits 部分。

#### 当你的 Credits 用完时会发生什么？
当你的 Credits 即将用尽时，你会在个人用量页面和 Qoder 内看到明确的提示。此时你可以选择升级到更高的订阅方案。如果你的 Credits 配额完全耗尽，你将自动切换到基础模型，基于基础模型服务仍会继续提供。基础模型设有每日使用上限；一旦达到上限，你需要等到次日才能继续使用。

如果你使用的是 Free、Pro Trial、Pro 或 Pro+ 方案，你可以随时升级到更高档位以获得更多 Credits 配额。如果你已在 Ultra 方案，请等待我们即将推出的资源附加包选项来提升配额。

#### Pro 试用规则
我们很高兴为首次登录 Qoder 客户端的新用户提供为期 14 天的免费 Pro 试用（需使用 Qoder 最新版本；通过非虚拟机方式登录），并附赠 300 个 Credits，便于体验我们的 Pro 方案功能。为确保所有用户都能获得公平且顺畅的体验，我们设定了合理的使用上限。本优惠每位用户限领一次，任何额外创建的试用账户将被冻结。

#### 升级订阅方案
您可随时升级订阅。当前方案中未使用的 Credits 将自动结转，不会丢失。请注意，一旦 Credits 结转完成，该操作不可撤销。

**从 Pro 试用版升级：** 若在试用期内升级，未使用的赠送 Credits 将自动以附加包形式转入您的账户，并保留原有到期日期。

**从 Pro 升级到 Pro+：** 若在订阅周期中从 Pro 升级到 Pro+，Pro 方案中未使用的 Credits 将自动以附加包形式转入您的账户，并保留原有到期日期。您的原 Pro 方案将立即失效，新 Pro+ 方案的订阅周期将开始计算。

因此，您可随时升级，无需担心未使用的 Credits 资源会丢失。

#### 降级您的订阅方案
您可随时取消订阅。当前方案将在本计费周期结束前继续保持有效，计费周期结束后您的账户将自动降为免费方案。请注意，任何未使用的 Credits 在降级后将过期且不可转移。

我们不支持在订阅周期内立即降级方案。

#### 高级功能
与 Free 方案相比，付费方案支持以下高级功能：

# 账户

## 计费

本文介绍 Qoder 的计费，并演示如何查看和管理您的订阅与账单。

### 管理你的套餐
1. 登录 Qoder 网站，点击右上角头像，进入 **Settings > Plan & Billing**
2. 在这里，你可以查看当前订阅套餐，并可一键升级至更高档位

### 账单
在账单部分，您可以查看完整的付款记录，并下载任意历史账单以供留存。

### 计费周期
我们的订阅按月计费。

您的套餐将于每月与最初订阅同一天自动续订。续订付款会在当前周期到期前一天处理。若续订付款失败，您下一周期的订阅将被取消，账户将降级为 Free Plan，且任何未使用的 Credits（可消耗额度）将被作废。

**计费周期示例：**

- **普通订阅情况：** 若您在 2025 年 8 月 16 日订阅，当前计费周期将于 2025 年 9 月 16 日 23:59:59 结束。
- **月末订阅情况：** 若您在 2025 年 2 月 28 日订阅，您的周期将于 2025 年 3 月 28 日 23:59:59 结束。若您在 2025 年 3 月 31 日订阅，您的周期将于 2025 年 4 月 30 日 23:59:59 结束（因 4 月无 31 日）。

### 支持的支付方式
我们目前支持 Visa、Mastercard 和支付宝。更多支付方式即将推出！

### 取消订阅
在 Plan & Billing 页面，找到 Plan 卡片并点击 **Manage > Cancel Subscription**。

请注意，取消订阅将在您当前的计费周期结束时生效。这意味着您的订阅不会自动续订。您的当前订阅将在到期日前保持继续有效。

### 账单常见问题
#### 为什么我的付款失败？

付款可能因多种原因失败，包括：

- **资金不足：** 您的账户余额不足以支付该费用。
- **卡片过期或无效：** 您用于付款的银行卡已过期，或卡片信息输入有误。
- **银行拒付：** 您的银行拒绝了此次付款请求。请联系银行了解更多信息。

#### 如何更新我的账单信息？
您可以进入 **Settings > Plan & Billing > Manage Billing Information**，在该页面更新账单地址和管理支付方式。默认付款方式将自动用于订阅计划的续订付费。此更改将适用于今后的所有购买与续订。

我们使用 Antom 进行安全的支付处理。请注意，变更仅影响后续交易，我们无法修改历史发票上的信息。

#### 如何申请退款？
请注意，退款申请须满足以下两个条件：在订阅后 24 小时内提出申请，且未使用过任何 Credits。

如需申请，请将您的 Qoder 账户和相关账单信息发送至 refund@qoder.com。Qoder 支持团队将尽快协助您处理。退款流程通常需要 5-10 个工作日。

#### 我还有其他账单问题。如何获得帮助？
如果您有此处未解答的账单问题，请发送电子邮件至 contact@qoder.com。

为更快获得支持，请使用与您账户关联的邮箱来信，并详细说明您的问题。
