# 通义灵码专属版安全体系介绍

## 一、产品概述

通义灵码为了应对中大型企业普遍的代码安全诉求推出了专属版方案。提供企业账号集成、统一授权管理、企业知识库问答、企业代码库生成增强等能力，满足企业安全合规要求及个性化使用场景的需求。

通义灵码专属版产品的核心特点是提供VPC独立实例部署，支持企业通过专线与阿里云VPC连接形成全内网模式，无公网访问入口，并且与其他企业部署实例网络隔离。同时还支持企业账号以及SSO集成，支持专属模型算力部署等高级特性。

本文将从系统架构、技术防护、安全认证等多维度介绍产品安全体系。

## 二、系统架构

通义灵码专属版是通过在云端VPC内独立部署的灵码服务实例，实现与客户内网进行打通，从而实现高度的安全性。既能满足客户使用公共云云推理资源的低成本、大参数模型诉求，又可以满足强代码安全保障。

### 通义灵码专属版架构特点：

1. 灵码服务在VPC内独立部署，与其他用户服务网络隔离。
2. 可通过专线、Private link实现企业内网与灵码VPC全内网打通，灵码服务不暴露任何公网地址。
3. 可设置允许访问的IP白名单，进一步保障访问安全。
4. 可实现专属数据存储，与其他客户物理级网络隔离。
5. 可实现代码、文档、向量存储放置于企业内网。云端完全不存储任何企业代码、文档数据。
6. 支持企业私有模型部署在专属推理集群中，实现专属模型服务网络隔离。

### 架构优势：

1. 强安全隔离性：全内网访问，网络层物理隔离。
2. 高稳定性保障：独立计算资源部署，不受其他客户影响。
3. 支持专属推理资源：支持微调训练后模型专属推理资源部署。
4. 高定制化：支持问答模型API扩展、敏感信息自定义扩展等。
5. 支持私有化特性：企业私有账号体系对接、支持企业内多租户管理。

## 三、技术防护

通义灵码专属版部署架构可以通过VPC独立部署保障安全性以外，设置多项技术方案保障企业数据安全。

1. **全链路代码加密：**
   通义灵码IDE插件会在客户端本地预处理代码，并实现代码加密，通过https协议加密传输给大模型推理，即使在传输过程中被劫持，也无法复原代码源文件。

3. **传输过程加密：**

4. **本地代码和向量存储：**
   工程级别向量存储均在用户本地环境上，不会上传到云端存储。企业级别代码和文档及其向量存储，可存储在企业内网服务器上，降低在云端泄露的风险。

5. **敏感信息过滤：**
   可以通过企业级配置，在用户IDE端进行敏感信息过滤。支持通过正则表达式过滤，支持接入企业安全平台，对所有上传信息进行过滤和审计，若发现不符合安全要求的内容，可以进行拦截处置或拦截替换。

6. **仅使用开源合规代码训练模型：**
   产品训练数据均来源于优质开源代码、技术社区文章和技术文档、阿里云的云服务文档、阿里云 SDK/API 使用文档等，不含任何企业私有数据。

7. **法律保障：**
   通义灵码承诺绝不会存储企业用户的代码，以及用于二次训练，因此不存在数据泄露可能。灵码需要上传代码片段到服务端进行推理（包括当前圈选代码片段、补全位置的上下文、工程关键信息索引、相似性代码片段等），过程中我们不会存储企业的代码数据，推理结束在内存中即被销毁，不存在泄露的可能。而且从技术上也无法用于二次训练。详情见官网通义灵码隐私政策。

## 四、安全权威认证

通义灵码产品严格按照阿里云安全体系构建，已通过了公安部网络安全等级保护 2.0 三级认证、ISO 27001 信息安全管理体系标准认证、ISO 9001 质量管理体系认证，第一批通过信通院代码大模型标准的厂商（4+级），标志着通义灵码的数据保密性、完整性、可用性和隐私性达到业内顶尖水平。
